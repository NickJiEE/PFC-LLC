# Issue 6: Switched-Current Logging Bias in Component-Stress Validation

## Problem

The logged switched-current signals produced a small DC-area error during steady-state component-stress validation.

The affected signals were primarily:

```text
iCout
iBoostDiode
```

The original steady-state results showed:

```text
Raw iCout average         = 0.011762649 A
Raw iBoostDiode average   = 0.261573833 A
Iout average              = 0.249811185 A
```

The current logs were internally consistent because:

```text
iBoostDiode average - Iout average
= 0.261573833 - 0.249811185
= 0.011762648 A
```

This matched the reported capacitor-current average.

However, a settled output capacitor should have an average current close to zero. The measured output voltage did not increase at the rate implied by an average capacitor current of approximately `11.76 mA`.

Using the output-voltage endpoints:

```text
Expected iCout average from C*dV/dt
= -0.000007965 A
```

The logged switched-current average was therefore inconsistent with the capacitor state.

---

## Cause

The current signals were generated by ideal or near-ideal switching elements and logged at irregular solver time steps.

At switching events, the logged data can contain abrupt transitions, repeated timestamps, and left-side or right-side event values.

Direct numerical integration can then introduce a small artificial area around each switching edge.

The error per switching event is very small, but the steady-state validation window contained more than twenty thousand switching cycles. The accumulated area appeared as a false DC component in the switched-current averages.

Replacing trapezoidal integration with uniform previous-value resampling did not remove the full error. This showed that the remaining bias was already present in the logged event data rather than being caused only by the post-processing method.

---

## Effect on the validation process

The issue mainly affected:

```text
Average output-capacitor current
Average boost-diode current
RMS values that included the false DC component
```

The following measurements were not materially affected:

```text
Peak current
Maximum voltage
Minimum voltage
Reverse-voltage stress
Gate pulse width
Switching frequency
Continuous inductor-current RMS
Source-current RMS
```

The output-capacitor RMS value changed only slightly after removing the false DC component.

```text
Raw iCout RMS        = 0.657184487 A
Corrected iCout RMS  = 0.657079212 A
```

The peak values remained unchanged.

Therefore, the measurement error did not invalidate the main peak-current or voltage-stress results, but it did require correction before using the switched-current averages and related RMS values in the final component-stress report.

---

## Correction approach

The correction used output-capacitor charge balance.

For settled operation, the expected average capacitor current is:

```text
Icap_expected = Cout * (Vout_end - Vout_start) / measurement_duration
```

Using:

```text
Cout = 220 uF
Measurement window = 4.8 s to 5.0 s
```

the expected capacitor-current average was:

```text
Icap_expected = -0.000007965 A
```

The inferred logging bias was:

```text
DC bias
= raw iCout average - expected iCout average
= 0.011762649 - (-0.000007965)
= 0.011770614 A
```

The corrected capacitor-current waveform estimate was:

```matlab
iCout_corrected = iCout_uniform - dc_bias;
```

This correction removed only the inferred DC-area offset. It did not change the measured switching-current peaks.

---

## Corrected output-capacitor result

After charge-balance correction:

```text
Corrected iCout average    = -0.000007965 A
Corrected iCout RMS        = 0.657079212 A
iCout ripple RMS           = 0.657079212 A
Measured iCout peak        = 3.11099 A
```

The corrected average is effectively zero, as expected for a settled capacitor.

The RMS value changed by only approximately:

```text
0.000105 A
```

Therefore, the original output-capacitor RMS stress was already close to the correct value.

The validated output-capacitor ripple-current stress is:

```text
0.657 A RMS
```

---

## Corrected boost-diode result

The boost-diode average was corrected using output-stage charge balance:

```text
IboostDiode_average
= Iout_average + Icap_expected
```

This gave:

```text
Corrected boost-diode average = 0.249803220 A
```

which closely matched:

```text
Iout average = 0.249811185 A
```

The corrected boost-diode values were:

```text
Raw boost-diode RMS        = 0.707344719 A
Bias-adjusted RMS estimate = 0.703077044 A
Measured peak current      = 3.36141 A
```

Because the actual logging error likely occurs only around switching edges rather than as a physically uniform DC offset, the raw RMS value was retained as the conservative component-rating value:

```text
Boost-diode RMS used for rating = 0.707 A
```

---

## Script changes

The MATLAB validation script was modified to:

```text
1. Remove duplicate timestamps while keeping the final event value.
2. Resample switched-current signals on a uniform 100 ns grid.
3. Use previous-value interpolation instead of linear interpolation.
4. Calculate raw averages and RMS values from the uniform samples.
5. Estimate the expected capacitor average from C*dV/dt.
6. Calculate the inferred DC-area bias.
7. Report corrected iCout average and RMS.
8. Report corrected boost-diode average and RMS estimate.
9. Preserve all directly measured peak values.
10. Save a separate charge-balance correction CSV.
```

The correction results were saved separately so the original measurements remained available for comparison.

---

## Validation rule after correction

For switched-current validation:

```text
Use measured peaks directly.
Use corrected capacitor average and ripple RMS.
Use charge-balanced boost-diode average.
Use the larger raw boost-diode RMS as the conservative rating value.
Do not use the uncorrected switched-current DC averages.
```

The correction avoids rerunning the full simulation while preserving conservative component-stress values.

---

## Limitation

The correction uses `Vout` to estimate capacitor charge balance.

Because `Vout` includes the capacitor ESR voltage, the ideal validation signal would be the voltage directly across the ideal capacitor element:

```text
VcapIdeal
```

A future model revision could log `VcapIdeal` and use:

```text
Integral of iCout dt = Cout * change in VcapIdeal
```

For the current validation window, the output voltage was settled and the window contained an integer number of line cycles, so the `Vout` endpoint method provided a useful correction without requiring another simulation.
